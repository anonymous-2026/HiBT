Abstract

Long-horizon robotic assembly requires planners to reason over object relations, tool use, and ordered preconditions while producing executable control structures. However, most LLM-based planners rely on flat token-by-token generation, which does not explicitly capture the dependencies required for Behavior Tree (BT) synthesis. We propose HiBT, which formulates BT synthesis as hierarchical concept planning. HiBT predicts hierarchical concepts from the task goal and symbolic world state, induces a planning interface through prototype-guided matching, applies deterministic realization and dependency repair, and compiles the resulting symbolic plan into a BT. On a 60-instance language-conditioned assembly benchmark using four open-source LLM backbones, HiBT achieves 60/60 executability, logical coherence, and task success for each backbone, while reducing generation time to 0.35-0.45 seconds. In LIBERO rollouts, HiBT reaches 89.2% final-goal success on LIBERO-Long with shorter rollouts, and 72.3% strict success on the dependency-intensive LIBERO-Hier suite, outperforming flat-instruction and action-sequence guidance. These results show that a dependency-aware hierarchical interface supports both executable BT synthesis and long-horizon embodied execution.

Hierarchical Concept Planning

HiBT separates high-level task reasoning from symbolic execution. A learned interface proposes task-compatible hierarchy, while deterministic realization closes tool, grasp, and ordering dependencies before BT compilation.

  • Predict: map task language and symbolic state to a fixed-geometry concept pyramid.
  • Decode: retrieve a structural template and slot-level symbolic candidates through prototype matching.
  • Repair: insert missing support actions for unmet preconditions.
  • Compile: turn grounded method nodes into an executable Behavior Tree.

Planning Interface

The example shows how HiBT turns a language-conditioned assembly request into a structured interface before any Behavior Tree is emitted. The task goal and initial symbolic state are first mapped into a hierarchical concept pyramid, then decoded against a prototype bank to recover a task-compatible planning template and slot-level symbolic candidates.

The realized plan is not treated as a flat action list. HiBT records the target predicate, stable subgoals, grounded method nodes, and an auxiliary action sequence. Dependency closure checks each grounded action against the current state and inserts support actions when tool, grasp, or ordering preconditions are missing.

The compiler operates on repaired method nodes and stable subgoals, so the generated BT contains explicit condition checks and reusable support branches. This is the main distinction from direct BT generation and action-sequence-first planning: hierarchy and dependency closure are made explicit before compilation.

  • Input: task language, symbolic initial state, target predicate, and action domain.
  • Interface: hierarchical concepts plus prototype-guided template and slot retrieval.
  • Repair: deterministic closure for missing support conditions.
  • Output: a dependency-aware BT, with the action sequence retained only for reporting and consistency checks.

Results

HiBT is evaluated on symbolic robotic assembly and embodied LIBERO rollouts. The main evidence is that hierarchical concepts plus deterministic repair produce fully executable and logically coherent BTs across model backbones, while the same interface improves long-horizon VLA execution.
